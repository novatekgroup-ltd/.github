# Security Policy

## Reporting a Vulnerability

At NovaTek Group Ltd, security is foundational to everything we do. We deeply appreciate the efforts of security researchers and the community in helping us maintain the highest standards.

### How to Report

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, send an encrypted email to our dedicated security team:

- **Email:** [security@novatekgroup.com](mailto:security@novatekgroup.com)
- **PGP Key:** [Download public key](https://novatekgroup.com/.well-known/security-pgp-key.asc) *(placeholder URL)*

Please include the following details in your report:

- A clear description of the vulnerability.
- Steps to reproduce or a proof‑of‑concept.
- Potential impact of the issue.
- Any suggested remediation (if available).

### What to Expect

| Timeline | Action |
|----------|--------|
| Within 24 hours | Acknowledgment of receipt |
| Within 72 hours | Initial triage and severity assessment |
| Within 10 business days | Status update and estimated fix timeline |
| Upon resolution | Coordinated disclosure (if mutually agreed) |

### Supported Versions

We provide security updates only for the latest release of any public‑facing software or API. Legacy versions are not supported.

| Version | Supported |
|---------|-----------|
| Latest (1.x) | ✅ Yes |
| < 1.0 | ❌ No |

### Responsible Disclosure

We are committed to working with researchers in good faith. We will not pursue legal action against individuals who:

- Follow this reporting policy.
- Avoid testing on production systems without explicit authorization.
- Do not disclose vulnerability details publicly before a fix is available.

### Hall of Fame

Researchers who report valid vulnerabilities may be eligible for acknowledgment in our Security Hall of Fame (with permission).
